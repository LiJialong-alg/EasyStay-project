# 酒店预订平台 - 产品需求文档 (PRD)

## 1. 产品概述

本产品是一个类似携程、美团、Airbnb 或 Booking 的综合性酒店预订管理平台。系统架构分为两个独立的管理端：**商家端 (Merchant Portal)** 和 **平台管理端 (Platform Admin Portal)**。

*   **商家端**：供入驻平台的酒店商家使用，用于管理其旗下的酒店房态、订单处理、财务对账及用户评价。
*   **平台管理端**：供平台运营方使用，用于监管所有商户、审核酒店资质、管理全平台用户订单及资金流向。

## 2. 核心用户角色

### 2.1 平台运营方 (Platform Admin)
*   **超级管理员**：拥有系统最高权限，负责平台整体配置、账号管理。
*   **运营专员**：负责商家入驻审核、酒店上架审核、评价监管。
*   **财务专员**：负责商户提现审批、平台佣金结算、财务报表查看。

### 2.2 酒店商家 (Merchant)
*   **商户管理员**：入驻平台的酒店拥有者，管理旗下所有酒店信息、财务账户。
*   **酒店前台/店长**：处理日常订单、房态维护、住客接待。

## 3. 功能模块详解

### 3.1 商家端 (Merchant Portal)

| 模块名称 | 功能描述 |
|----------|----------|
| **控制台 (Dashboard)** | 展示今日订单数、即将入住/离店、实时房态概览、核心经营数据（营收、入住率）。 |

| **评价管理** | • **评价列表**：查看用户对酒店的评分和评论。<br>• **回复评价**：商家对评论进行公开回复。 |
| **财务中心** | • **资金流水**：查看每笔订单的收入明细。<br>• **提现管理**：绑定银行卡、申请提现、查看提现进度。<br>• **对账单**：月度/季度财务报表下载。 |

### 3.2 平台管理端 (Platform Admin Portal)

| 模块名称 | 功能描述 |
|----------|----------|

## 4. 核心业务流程

### 4.1 商家入驻与上架流程
1.  商户注册账号并提交资质信息（营业执照等）。
2.  **平台运营**审核商户资质 -> 通过/驳回。
3.  商户登录商家端，创建酒店和房型，提交上架申请。
4.  **平台运营**审核酒店信息（图片、价格、设施） -> 通过/驳回。
5.  审核通过后，酒店在 C 端上线，开始接单。

### 4.2 订单流转流程
1.  用户 (C端) 下单并支付。
2.  **商家端**收到“待确认”订单通知。
3.  商家确认接单 -> 订单变为“待入住”；(若拒单，系统自动退款)。
4.  用户到店，商家办理入住 -> 订单变为“已入住”。
5.  用户离店，商家办理退房 -> 订单变为“已完成”。
6.  订单金额（扣除佣金后）进入商户余额，可申请提现。

### 4.3 提现流程
1.  商户发起提现申请。
2.  资金冻结。
3.  **平台财务**审核申请 -> 确认打款。
4.  资金扣除，状态变更为“提现成功”。

## 5. 用户界面设计 (UI)

### 5.1 设计风格
*   **风格定位**：高效、清晰、企业级 (Enterprise SaaS)。避免过度装饰，强调数据可读性。
*   **配色方案**：
    *   **商家端**：以**深蓝/品牌蓝**为主色，体现专业与信任。
    *   **管理端**：以**深紫/暗黑**为主色，体现监管与权限的区分。
*   **布局结构**：
    *   左侧侧边栏导航 (Sidebar Navigation)。
    *   顶部通栏 (Top Header) 显示用户信息和全局操作。
    *   卡片式内容区域 (Card-based Content Area)。

### 5.2 关键页面交互
*   **房态日历**：支持拖拽选择日期范围，右键菜单进行批量开关房操作。
*   **房态看板 (新)**：高密度网格视图，使用颜色块区分（红=满，绿=空，黄=脏），悬停显示详情，点击弹出操作菜单。
*   **数据大屏**：使用 ECharts 展示动态折线图/柱状图，支持时间范围筛选。
*   **审核列表**：采用左右分栏或抽屉式 (Drawer) 详情页，方便在不离开列表的情况下快速审核。

## 6. 非功能性需求

*   **多租户隔离**：严格的数据权限控制，商户A绝不可看到商户B的数据。
*   **高并发支持**：节假日高峰期订单处理的稳定性。
*   **数据一致性**：确保订单状态在 C 端、商家端、管理端三方实时同步。
*   **审计日志**：记录所有关键操作（如修改价格、强制退款、账号封禁）的操作人、时间和IP。
